之前我们爬取的都是本地自己制作的一些简单网页，接下来我们尝试对真实世界的网页进行爬取。
今天的主题是猫途鹰网站：”https://cn.tripadvisor.com/“

1 背景介绍

首先我们要了解的是服务器与本地的交换机制，大致就是我们个人PC向服务器发送一个Requests，服务器收到这个Requests后，会对我们做出回应，返回一个Response。
这一步我们的关注点在Requests上面，在Http1.1的时代，Request请求目前有八种方法：Post,Get,Head,Put,Options,Connect,Trace,delete。但是最常用的仍然只有Post和Get，而我呢用Python爬取信息的时候，其实就是模拟的这两种方法。

2 网站爬取

2-1 观光景点列表的爬取

从下图中我们可以看到，纽约市的各大著名景点，然后每个经典标识有图片，有标题，有评分，有标签，我们首先就是要爬取这些内容。

我们可以看到下图的最后一图中是由四张小图构成的，这种类似景点集合的图片我们不准备搜集，所以在采集信息的时候要注意区分。

代码如下所示：

from bs4 import BeautifulSoup
import requests

url = 'https://cn.tripadvisor.com/Attractions-g60763-Activities-New_York_City_New_York.html'
wb_data = requests.get(url)

soup = BeautifulSoup(wb_data.text,'lxml')

#采集图片地址，名称，标签和评星状况

images = soup.select('div.photo_booking.non_generic > a > img[width="160"] ')#图片暂时抓不到，因为有网站的反爬取，我们可以尝试抓取移动端
titles = soup.select('div.property_title > a[target="_blank"]')
cates = soup.select('div > div.element_wrap > div > div.p13n_reasoning_v2 ')
stars = soup.select('div > div > span.rate.rate_no > img')

for image,title,cate,star in zip(images,titles,cates,stars):

    data = {
            'title' : title.get_text(),
            'image' : image.get('src'),
            'cate' : list(cate.stripped_strings),
            'star' : star.get('alt')
            }
    print(data)

在爬取图片的时候，我们发现，图片集合中的图片的宽是75，而别的非集合的图片的宽和高都是160，所以我们可以用图片属性进行区分。另外在标题中，非集合的图片的连接会有一个target = "_blank"属性，用来指明点击后新网页的打开方式，这个也和集合类的有区别，所以，从这两点下手，我们可以在结果中剔除集合类的图片信息。

2-3 爬取全部页面的景点信息

def scrapy_list(url,data=None):
    response = requests.get(url)
    soup = BeautifulSoup(response.text,'lxml')
    time.sleep(4) #为了防止被网站封IP，我们可以设置每4秒爬取一次
    images = soup.select('div.photo_booking.non_generic > a > img[width="160"] ')#图片暂时抓不到，因为有网站的反爬取，我们可以尝试抓取移动端
    titles = soup.select('div.property_title > a[target="_blank"]')
    cates = soup.select('div > div.element_wrap > div > div.p13n_reasoning_v2 ')
    stars = soup.select('div > div > span.rate.rate_no > img')

    for title,cate,image,star in zip(titles,cates,images,stars):
        data = {
                'title' : title.get_text(),
                'cate' : list(cate.stripped_strings),
                'image' : image.get('src'),
                'star' : star.get('alt')
                }
        print(data)

def scrapy_collect(url,data=None):
    response = requests.get(url,headers=headers)
    soup = BeautifulSoup(response.text,'lxml')

    titles = soup.select('span > div.cvs-content > div.info > a[class="location-name"]')
    images = soup.select('div.cvs-content > div.photo > div.sizedThumb > img')#存储中的图片并没有反爬取处理，所以我们可以正常的爬取
    stars = soup.select('div.cvs-content > div.info > div > span > img')
    addresses = soup.select('div.cvs-content > div.info > span')

    for title,image,star,address in zip(titles,images,stars,addresses):
        address_result = list(address.stripped_strings)
        str = ' '
        address = str.join(address_result)

        data = {
                'title' : title.get_text(),
                'image' : image.get('src'),
                'star' : star.get('alt'),
                'address' : address
                }
        print(data)

for single_url in urls:
    scrapy_list(single_url)

2-4 简易反爬取措施

一般情况下网站的PC端都会比移动端的反爬取措施要严格，因为移动端要适配许多不同的手机型号，所以我们在遭受PC的反爬取情况下，可以尝试区爬取移动端的内容。
同样还是用Google Chrome来进行这项操作：

观察图中，我们选定的iphone6的型号进行适配，然后从Network中获取User-Agent的信息以便于伪装成移动端。

程序代码如下：

from bs4 import BeautifulSoup 
import requests

headers = {
    'User-Agent' : 'Mozilla/5.0 (iPhone; CPU iPhone OS 9_1 like Mac OS X) AppleWebKit/601.1.46 (KHTML, like Gecko) Version/9.0 Mobile/13B143 Safari/601.1'
}
url = "https://cn.tripadvisor.com/Attractions-g60763-Activities-New_York_City_New_York.html#ATTRACTION_LIST"

response = requests.get(url,headers=headers)
soup = BeautifulSoup(response.text,'lxml')

# images = soup.select('div.thumb.thumbLLR.soThumb > img') #开始我尝试直接爬取img信息，结果同样被反爬了，于是仔细观察图片处的代码，我们可以爬取旁边的div的属性
images = soup.find_all("div","missing lazyMiss")
titles = soup.select('div.container.containerLLR > div.title.titleLLR > div')

for image,title in zip(images,titles):

        'title' : title.get_text()
    }
    print(data)

这个在最初爬取时，还是会被反爬，所以失败了好多次，然后我们仔细观察图片处的代码：

会发现div下有div和img两个兄弟结点，而这两个兄弟结点中，img的src属性的内容与div的data-thymburl属性的内容相同，而且正好是我们所需要的图片地址链接，于是，我们可以放弃直接爬取img的src，转而爬取div.missing lazyMiss的data-thumburl属性的内容。然后完美的得到了想要的结果。
