背景

我们在爬取58的时候，会发现二手手机号码页面与其他的页面完全的不同，整个页面是由一个个的手机号码组成，所以，为了完整的爬取58的内容，我们需要对手机号码的页面进行单独的爬取。

我们首先要爬的就是这个充满标题与链接的页面: ，这里面我们要注意不能爬取商家的链接，因为商家的链接基本都指向了不同架构的页面，而且并不具备二手商品的爬取价值，所以我们不做考虑。那么如何区分商家和个人呢，我们发现，商家推广的方框右上角会有一个数字的角标，而个人的并没有，我们就可以根据这个来做判定。这部分我们可以写一个函数：

urls = ['http://bj.58.com/shoujihao/pn{}'.format(str(i)) for i in range(1,100)]

def get_phone_no(url):

    resp = requests.get(url)
    soup = BeautifulSoup(resp.text,'lxml')
    # time.sleep(2) 

    titles = soup.select('#infolist > div.boxlist > ul > div.boxlist > ul > li > a.t > strong.number')
    links = soup.select('#infolist > div.boxlist > ul > div.boxlist > ul > li > a.t')

    for title,link in zip(titles,links):
        data = {
            'title':title.get_text(),
            'link':link.get('href').split('?')[0]
        }
        # 这里我们要桌一个判断，如果获取的链接是这种跳转到外部链接的商家链接，就不往数据库中存
        if data['link'] == 'http://jump.zhineng.58.com/clk':
            pass
        else:
            sheet_phone_url.insert_one(data)
            print(data)

这样就成功的获取了100页内的所有的个人的手机号的链接和标题，然后我们再写一个用于爬取详情页的函数，将这里面爬取到的链接传入这个函数，即可爬取到所有链接的详情页：

def get_phone_detail(url):

    resp = requests.get(url)

    data = {
        'title': soup.select('div.col.detailPrimary.mb15 > div.col_sub.mainTitle > h1')[0].get_text().replace('\n','').replace(' ','').replace('\t',''),

        'area':soup.select('ul.suUl > li:nth-of-type(2) > div.su_con > a')[0].get_text() if soup.find(text='区域：') else None,
        'host':soup.select('ul.userinfo > ul.vcard > li > a')[0].get('title'),

    }
    try:
        sheet_phone_detail.insert(data)
    except pymongo.errors.DuplicateKeyError as ex:
        pass
    print(data)

总结

首先在爬取页面部分，并没有遇到太大的困难，和之前的步骤基本相同，这里加入的新东西有这么几个，首先就是数据库的重复检测，然后是多进程。

1-数据库重复检测

看这两句代码：

sheet_phone_url.ensure_index('link',unique=True,dropDups=True)
sheet_phone_detail.ensure_index('link',unique=True,dropDups=True)

这句是对数据库中的数据进行重复检测，这里最好是采用唯一的值，比如我们这个程序中的网址，每个项目的网址肯定是唯一的，然后dropDups=True是保证若有重复的项插入时，删除重复的数据，若是不加入这句的话，当插入重复数据时程序会报错。
当然，为了是程序更加健壮，可以再加入一个try..except..：

try:
    sheet_phone_detail.insert(data)
except pymongo.errors.DuplicateKeyError as ex:
    pass

2-多进程

以下内容参考自廖学峰Python教程，感谢作者

我们简单介绍下进程和线程。
对于操作系统来说，一个任务就是一个进程，比如开一个浏览器就是启动一个浏览器进程，开一个word就是启动一个word进程。有些进程还不止同时干一件事，比如Word，它可以同时进行打字、拼写检查、打印等事情。在一个进程内部，要同时干多件事，就需要同时运行多个“子任务”，我们把进程中的这些“子任务”称为线程。

2-1 multiprocessing模块

要让Python程序实现多进程(Multiprocessing)，我们先来看一下multiprocessing模块。
multiprocessing模块提供了一个Process类来代表一个进程对象，下面例子演示了启动一个子进程并等待其结束：

from multiprocessing import Process
import os

# 子进程要执行的代码
def run_proc(name):
    print('Run child process %s (%s)...' % (name, os.getpid()))

if __name__=='__main__':
    print('Parent process %s.' % os.getpid())

    #创建Process实例
    p = Process(target=run_proc, args=('test',))
    print('Child process will start.')
    #启动      
    p.start()
    p.join()
    print('Child process end.')

运行结果如下：

Parent process 928.
Process will start.
Run child process test (929)...
Process end

创建子进程时，只需要传入一个执行函数和函数的参数，创建一个Process实例，用start()方法启动，这样创建进程比fork()还要简单。join()方法可以等待子进程结束后再继续往下运行，通常用于进程间的同步。

2-2 Pool进程池

如果要启动大量的子进程，可以用进程池的方式批量创建子进程:

from multiprocessing import Pool
import os, time, random

def long_time_task(name):
    print('Run task %s (%s)...' % (name, os.getpid()))
    start = time.time()
    time.sleep(random.random() * 3)
    end = time.time()
    print('Task %s runs %0.2f seconds.' % (name, (end - start)))

if __name__=='__main__':
    print('Parent process %s.' % os.getpid())
    p = Pool(4)
    for i in range(5):
        p.apply_async(long_time_task, args=(i,))
    print('Waiting for all subprocesses done...')
    p.close()
    p.join()
    print('All subprocesses done.')

运行结果：

Parent process 4002.
Waiting for all subprocesses done...
Run task 0 (4003)...
Run task 1 (4004)...
Run task 2 (4005)...
Run task 3 (4006)...
Task 2 runs 0.70 seconds.
Run task 4 (4005)...
Task 3 runs 1.45 seconds.
Task 1 runs 1.91 seconds.
Task 4 runs 1.87 seconds.
Task 0 runs 2.85 seconds.
All subprocesses done.

对Pool对象调用join()方法会等待所有子进程执行完毕，调用join()之前必须先调用close()，调用close()之后就不能继续添加新的Process了。

请注意输出的结果，task 0，1，2，3是立刻执行的，而task4要等待前面某个task完成后才执行，这是因为Pool的默认大小在我的电脑上是4，因此，最多同时执行4个进程。这是Pool有意设计的限制，并不是操作系统的限制。如果改成：

p = Pool(5)

就可以同时跑5个进程。由于Pool的默认大小是CPU的核数，如果你不幸拥有8核CPU，你要提交至少9个子进程才能看到上面的等待效果。
