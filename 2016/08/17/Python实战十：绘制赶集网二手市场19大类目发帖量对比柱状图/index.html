1、前情提要

从这一部分开始，我们要进行的是数据统计和分析部分，主要就是对之前我们爬取的数据做一个可视化的处理，使其更具有可看性，更容易去发现数据的价值。
开始之前，我们首先需要安装这么两个东西：

  • 安装网页编辑器Jupyter Notebook（其实就是之前的Ipython Notebook）
  • 安装数据可视化的库 Highchart

2、安装

2.1 Jupyter Notebook安装

很简单，直接使用pip即可，因为我是Mac，所以想要安装到Python3版本中的话，需要

pip3 install jupyter

安装好之后，我的是在命令行输入：

python -m IPython notebook

就会跳转到jupyter的网页，上面会显示我本地的文件，这样就说明安装成功了。

2.2 Highchart安装

同样，这个我们也是采用pip进行安装:

pip3 install charts

安装完成后，打开Jupyter，点击new > python3，然后在新页面中输入import charts，然后按crtl + 回车，若是出现类似这种结果：Server running in the folder /Users/wangcheng at 127.0.0.1:54485 则代表安装成功。
其他一些具体问题或者图表的问题，我们可查询Highchart 的官方网站

3、绘制赶集网二手市场各大类目发帖量对比的柱状图

上周完成的最后一项任务就是爬取赶集网二手市场的十万数据（其实爬取完发现并不足十万，一是其中有四个类目的不规则页面，懒得一一设计了，二是赶集貌似只存留一段时间内的数据，超过时间的帖子都被删除了），这周，将尝试着对这些数据进行处理，首先就是统计一下其各大类目的发帖量的多少。

import pymongo

然后我们要找的是其中的class_name的值，思路就是先将所有的class_name放入一个列表，然后置入一个数组进行去重，这样就可以知道一共有几个大类，再用类别进行索引：

class_list = []
for i in sheet_detail.find():
    class_list.append(i['class_name'])

class_index = list(set(class_list))
print(class_index)

这样就获得了类别的索引：

['二手手机', '家用电器', '图书/音乐/运动', '母婴/儿童用品', '二手家具', '二手笔记本', '农用品', '美容护肤/化妆品', '虚拟物品', '家居百货', '闲置礼品', '电子数码', '老年用品', '设备/办公用品', '服饰/箱包', '台式电脑/网络']

然后获取每个类别有多少个帖子：

class_count = []
for index in class_index:
    class_count.append(class_list.count(index))
print(class_count)

得到结果：

[1548, 1534, 1548, 1575, 1535, 1327, 1186, 1534, 1889, 1539, 3761, 1536, 1290, 3766, 1539, 1535]

最后开始准备构建生成图表所需的内容：

def get_chart(types):
    length = 0
    if length < len(class_index):
        for index,num in zip(class_index,class_count):
            data = {
                'name':index,
                'data':[num],
                'type':types
            }
            yield data
            length += 1

这里边主要就是构件一个字典，让它持续不断的返回字典中的内容，所以这里用了一个生成器，然后字典中的key值是Highchart规定的，不能更改。

series = [i for i in get_chart('column')]
charts.plot(series,show='inline',options=dict(title=dict(text='赶集网二手市场十六大类目发帖量比较')))

4、总结

4.1 难点一：Highchart导入时，有时会出现这种错误：

ImportError: No module named 'plot'

这时候我们要将本机中的charts文件夹中的文件进行替换，替换文件可以在麻瓜编程的Github中寻找，替换完成后再重新导入charts即可。

4.2 难点二：索引，每个类目发帖量的获取

这部分其实关键点就两个：

  • set的使用.
  • count的使用

观察之前代码即可了解这两个方法的应用情况，注意练习即可。
